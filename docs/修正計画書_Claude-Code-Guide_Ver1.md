## 2. 修正方針

### 2.1 現状と修正後の比較

| 項目 | 現状（Ver1） | 修正後（Ver2） |
|------|-------------|----------------|
| **セキュリティ** | Slack Token平文保存、--dangerously-skip-permissionsの危険性説明不足 | macOS Keychain使用、具体的な危険例を追加 |
| **構成整合性** | slides-outline.mdと実ドキュメントが不一致 | Section 1/9/10の対応ドキュメント作成 |
| **読者設定** | 初心者向けと謳いながら中上級者向け情報密度 | README.mdに「想定読者」セクション追加 |
| **Why補強** | 技術情報中心でBefore/After不明瞭 | 各ドキュメント冒頭にPain Point → Solution形式を追加 |
| **AI生成感** | ヒューマンスコア3/10、テーブル多用、定型フレーズ | テーブル50%散文化、個人体験談追加、定型フレーズ削減 |
| **ドキュメント構成** | usage-limits-sandbox.mdが3トピック混在 | security-guide.md / context-management.md / usage-limits.md に分割 |

### 2.2 修正の基本コンセプト

**セキュリティファースト × 実用性 × ヒューマンタッチ**

1. **セキュリティ最優先**: Critical 4項目を最優先で対応し、初心者が誤った設定をしない環境を構築
2. **構造的整合性**: アウトラインと実ドキュメントの一致、重複削除、ファイル分割による保守性向上
3. **ヒューマンライズ**: AI生成感を払拭し、実体験ベースの説明・個人的な失敗談を織り交ぜた読みやすい文章へ
4. **実用的な図解**: フローチャート・スクリーンショット追加で視覚的理解を促進
