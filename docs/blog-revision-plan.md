## 修正計画（優先度順）

### P0: 技術的正確性の修正（必須・即対応）

#### P0-1. コンテキスト色閾値の内部矛盾を解消
- **箇所**: L413-414（セクション6）vs L469（セクション7）
- **問題**: セクション6は「黄60-80%/赤80-100%」、セクション7は「黄70-85%/赤85%〜」と数値が異なる
- **対応**: 2つは別指標（コンテキスト使用率 vs 利用制限使用率）のため、それぞれ何の指標かを明記して区別する
- **工数**: 15分

#### P0-2. --dangerously-skip-permissions の安全条件を補完
- **箇所**: L279-281
- **問題**: 安全条件が2条件のみ（元は3条件: サンドボックス有効 + Denyリスト整備 + バックグラウンド実行が必要な場合に限定）
- **対応**: 第3条件を追記。L127のctコマンド説明にも「サンドボックス必須」を明示
- **工数**: 30分

#### P0-3. 権限モード名の正確性を確認・修正
- **箇所**: L222-227
- **問題**: 本文の `default/acceptEdits/plan/dontAsk/bypassPermissions` とソースドキュメントの `ask/auto/plan/delegate` が乖離
- **対応**: 公式ドキュメントで現行の正確なモード名を確認し統一
- **工数**: 30分

### P1: 信頼性・安全性の改善

#### P1-1. curl | bash にセキュリティ注意書きを追加
- **箇所**: L76-78
- **問題**: セクション4でパイプ攻撃の危険性を説明しながら、インストール手順には警告なし
- **対応**: コマンド直後に「公式URLであることを確認の上、実行してください」の1行追加
- **工数**: 15分

#### P1-2. AGENTS.md「Linux Foundation管理」の根拠確認
- **箇所**: L207
- **問題**: 一次情報源の参照がない
- **対応**: 公式ソースを確認し、出典リンクを追加。確認不可なら「〜とされている」に変更
- **工数**: 30分（調査含む）

#### P1-3. 冒頭にTL;DRと読者別誘導パスを追加
- **箇所**: L7-11の直後
- **問題**: 700行超の記事に要約がなく、読者別の推奨読み方もない
- **対応案**:
  ```
  ## TL;DR
  - Claude Codeは「タスクを委譲するAI」。Cursorの「補完するAI」とは設計思想が異なる
  - セットアップ → CLAUDE.md → 権限設定の3ステップで基本は完了
  - 今日から始めるなら: インストール → /init → /clear を習慣に

  | あなたの状況 | おすすめセクション |
  |---|---|
  | Claude Codeを触ったことがない | 1→2→12 |
  | インストール済みだが活用できていない | 3→4→6 |
  | Cursorから移行を検討中 | 1→10→3 |
  | 既に使い込んでいて効率化したい | 5→7→8→9 |
  ```
- **工数**: 30分

#### P1-4. MCP死んだ説の論理的接続を補足
- **箇所**: L500-531
- **問題**: MCPの課題を指摘した直後におすすめ6選を紹介し、判断基準が不明瞭
- **対応**: 引用直後に「それでもSlack/GitHub連携は実用的。必要なサーバーだけ厳選して使えば問題ない」という橋渡し文を追加
- **工数**: 15分

### P2: 品質向上（推奨）

#### P2-1. 地の文にヒューマンタッチを追加 [ユーザー判断]
- **箇所**: 記事全体の地の文
- **問題**: ヒューマンスコア5.0/10。セクション構造が均一、文末が「〜です」の連続、書き手の主観が不在
- **対応案（3段階）**:
  - A) 最小: リード文(L9-11)とまとめ(L716)のみ書き換え → 工数30分
  - B) 中: 各セクション冒頭に書き手コメント1文追加 + 連続引用の整理 → 工数2時間
  - C) 最大: 全セクションの地の文リライト → 工数4時間
- **推奨**: B（コスト対効果が最も高い）

#### P2-2. 長段落・長文の分割
- **箇所**: L116-119（tmux、150文字超）、L716（まとめ、220文字の1文）
- **対応**: 1段落1トピック原則で分割
- **工数**: 30分

#### P2-3. 連続引用の整理
- **箇所**: L44-48（3件連続）、L501-506（3件連続）
- **対応**: 引用は2件以内に絞り、残りは地の文に吸収
- **工数**: 30分

#### P2-4. 参考リンクURLの実在性確認
- **箇所**: L699-704
- **問題**: `code.claude.com` ドメインが正確か不明
- **対応**: 各URLのアクセス確認、正しいURLに修正
- **工数**: 15分

#### P2-5. 画像alt属性の改善
- **箇所**: 全52枚の `![...](...)`
- **問題**: alt属性がスライドタイトルのみ。画像非表示環境で情報が欠落
- **対応**: 各画像に内容を示す具体的なalt属性を設定
- **工数**: 1時間
