# Claude Code チーム導入ガイド

## はじめに

### こんな困りごとありませんか？

- 個人では便利だけど、チームに広げるときの進め方がわからない
- 導入コストやROIを上司にどう説明すればいいか迷っている
- メンバーのスキル差があり、全員が同じように使いこなせるか不安

### このガイドで解決できます

段階的な導入ロードマップに沿って進めれば、チーム全体の生産性を底上げしつつ、セキュリティリスクも管理できます。

| Before | After |
|--------|-------|
| 一部の人だけが使い、ノウハウが属人化 | 導入ロードマップで全員が段階的に習熟 |
| ROIが不明で予算承認が下りない | 定量指標で効果を可視化し、経営層に説明可能 |
| セキュリティが心配でチーム利用を躊躇 | 設定テンプレートで安全なベースラインを一括適用 |

### 前提条件

| 項目 | 要件 |
|------|------|
| アカウント | Claude Teams / Enterprise（チーム利用にはTeams以上を推奨） |
| 導入判断者 | テックリード or エンジニアリングマネージャー |

## 1. ROI（投資対効果）

### コスト構造

Claude Codeの利用コストは、プランの月額料金と、チームメンバーの習熟に必要な時間の2つです。

**プラン別の月額費用（1人あたり）**:

| プラン | 月額 | 主な対象 |
|--------|------|---------|
| Pro | $20 | 個人利用、試験導入 |
| Max 5x | $100 | 日常的に使うエンジニア |
| Max 20x | $200 | ヘビーユーザー、リードエンジニア |
| Teams | $30/人 | チーム利用（管理機能付き） |
| Enterprise | 要相談 | 大規模組織、SSO・監査ログ付き |

**習熟コスト**:
- 基本操作（CLAUDE.md、権限設定）: 1〜2時間
- 実務での定着（コンテキスト管理、Hooks活用）: 1〜2週間
- 上級活用（MCP、Agent Teams、カスタムスキル）: 1〜2ヶ月

### 期待効果

導入効果はタスクの種類によって異なりますが、以下の領域で特に効果が出やすいです。

**定型タスクの自動化**:
- ボイラープレートコードの生成: 手動30分 → Claude Code 5分
- テストコードの作成: 手動1〜2時間 → Claude Code 15〜30分
- ドキュメント作成・更新: 手動1時間 → Claude Code 15分

**品質向上**:
- コードレビューの事前チェック: Hooks + Lint自動実行で指摘事項を事前に解消
- セキュリティチェック: protect-sensitive-files.sh で機密ファイル保護を自動化
- 命名規則・フォーマットの統一: CLAUDE.mdでルールを定義し、全メンバーに適用

**ROI試算の目安**:
5人チームでTeamsプラン（$30/人/月 = $150/月）を使う場合、1人あたり月4時間の工数削減（時給$50換算で$200/人）が見込めれば、チーム全体で月$850のプラス効果です。実際には、習熟が進むにつれて削減幅はさらに大きくなります。

## 2. 習熟ロードマップ

チームメンバーのスキルレベルに合わせた段階的な習熟計画です。

### Week 1: 基礎固め（全員必須）

| 項目 | 内容 | 所要時間 |
|------|------|---------|
| インストール・認証 | Claude Codeのセットアップ、VSCode拡張 | 30分 |
| CLAUDE.md作成 | プロジェクト共通のCLAUDE.mdを作成・共有 | 1時間 |
| 権限設定 | チーム共通のsettings.jsonテンプレートを配布 | 30分 |
| 基本操作 | 簡単なタスク（ファイル検索、コード生成）を体験 | 1時間 |

### Week 2-3: 実務適用

| 項目 | 内容 | 所要時間 |
|------|------|---------|
| コンテキスト管理 | /clear、/compact の使い分け習得 | 30分 |
| 制限の把握 | ステータスラインの読み方、モデル切替 | 30分 |
| 実タスクでの利用 | 日常業務の一部をClaude Codeで実施 | 日常的 |
| 振り返り | チームで使い方を共有、困りごとを解消 | 1時間/週 |

### Week 4+: 応用・最適化

| 項目 | 内容 | 対象者 |
|------|------|--------|
| Hooks導入 | チーム共通のHooks（Lint、Push前チェック）を設定 | リードエンジニア |
| MCPサーバー | Slack、GitHub連携を設定 | リードエンジニア |
| カスタムスキル | チーム固有のワークフローをスキル化 | 上級者 |
| Agent Teams | 複数エージェントの協調作業 | 上級者 |

## 3. 導入ロードマップ

### Phase 1: パイロット（1〜2週間）

**目的**: 小規模に試して効果と課題を把握する

- 参加者: 2〜3名（技術リード + 意欲的なメンバー）
- プラン: 個人Pro or Max（低コストで試験）
- 成果物: 利用レポート（効果実感、課題リスト、推奨設定）

### Phase 2: チーム展開（2〜4週間）

**目的**: チーム全員に展開し、共通設定を整備する

- 参加者: チーム全員
- プラン: Teams（管理機能を活用）
- 成果物:
  - チーム共通CLAUDE.md
  - settings.jsonテンプレート（Allow/Deny/サンドボックス設定済み）
  - Hooks設定（protect-sensitive-files.sh、Lint自動チェック）
  - 習熟度チェックリスト

### Phase 3: 最適化・定着（1〜2ヶ月）

**目的**: 運用を最適化し、継続的な改善サイクルを回す

- MCPサーバー連携の拡充
- カスタムスキルの開発・共有
- 月次のROIレビュー（定量指標の計測）
- ナレッジベースの整備（FAQ、Tips集）

## 4. セキュリティガイドライン（チーム向け）

### チーム共通settings.jsonテンプレート

```json
{
  "permissions": {
    "defaultMode": "auto",
    "allow": [
      "Bash(git *)",
      "Bash(npm *)",
      "Bash(node *)",
      "WebSearch"
    ],
    "deny": [
      "Edit(.env)",
      "Edit(.env.*)",
      "Write(.env)",
      "Write(.env.*)",
      "Read(.env)",
      "Read(.env.*)",
      "Edit(credentials.json)",
      "Write(credentials.json)",
      "Read(credentials.json)",
      "Bash(rm -rf /)",
      "Bash(rm -rf /*)",
      "Bash(sudo *)"
    ]
  },
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true
  }
}
```

**ポイント**:
- `defaultMode: "auto"` でバランス型からスタート
- 機密ファイルへのアクセスをDenyリストで一律ブロック
- サンドボックスを有効にしてOS レベルの隔離を確保
- 必要に応じてプロジェクトごとにAllow設定を追加

### セキュリティチェックリスト

- [ ] settings.jsonのDenyリストに機密ファイルパターンを登録
- [ ] サンドボックスを有効化（sandbox.enabled: true）
- [ ] protect-sensitive-files.sh Hookを全メンバーに配布
- [ ] .gitignoreに`.env`、`credentials.json`等を登録
- [ ] 公開リポジトリにsettings.jsonをコミットしないルールを周知
- [ ] `--dangerously-skip-permissions`の使用ルールを定義

## 5. 効果測定の指標

### 定量指標

| 指標 | 計測方法 | 目標値 |
|------|---------|--------|
| タスク完了時間 | タスク管理ツールでBefore/After比較 | 20〜40%短縮 |
| コードレビュー指摘数 | PR当たりの指摘件数の推移 | 30%減少 |
| テストカバレッジ | CI/CDレポート | 10%向上 |
| ボイラープレート工数 | 定型タスクの所要時間 | 50%以上短縮 |

### 定性指標

| 指標 | 計測方法 |
|------|---------|
| メンバー満足度 | 月次アンケート（5段階評価） |
| 利用頻度 | 週あたりのセッション数 |
| 習熟度 | チェックリスト達成率 |
| ナレッジ共有 | Tips投稿数、FAQ更新頻度 |

## TIPS

- **パイロットは「やりたい人」から始める** — 強制導入は抵抗を生む
- **最初の1週間は設定サポートに時間を割く** — セットアップの躓きが離脱の最大原因
- **CLAUDE.mdはチームで育てる** — 全員が使う中で改善点を見つけて更新する
- **週次の共有会を設ける** — 「こんな使い方が便利だった」を共有するだけで習熟が加速する
- **制限の管理はリード層が把握する** — Teamsプランでの制限配分を意識的に管理する

---

## まとめ

チーム導入は「パイロット→チーム展開→最適化」の3段階で進めるのが最も低リスクです。最初の1〜2週間でROIの手応えを得られれば、チーム展開の推進力になります。セキュリティ設定はテンプレートで一括適用し、習熟はロードマップに沿って段階的に。全員が同じレベルを目指す必要はなく、基本操作ができれば日常業務で十分に効果が出ます。
