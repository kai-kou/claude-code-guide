## カラム定義

| カラム | 説明 |
|--------|------|
| TRY-ID | `TRY-{連番3桁}`形式（例: TRY-001）。全プロジェクト横断で一意 |
| 登録日 | レトロスペクティブでTryが特定された日付（YYYY-MM-DD） |
| ソース | Tryが生まれたスプリント（例: SPRINT-001）とプロジェクト名 |
| 改善内容 | 「何を」「どう改善」の形で記述 |
| 対象 | Rule / Skill / Subagent / Process / Template のいずれか |
| 優先度 | High / Medium / Low |
| 着手スプリント | In Progressに移行したスプリントID |
| タスクID | tasks.mdに登録された対応タスクのID |
| 効果 | 改善の実際の効果。定性的でもよい |
| キャンセル理由 | 棚卸しでキャンセルした理由 |

## ライフサイクル

```
[レトロスペクティブ]
    │ TRY-ID付与
    ▼
  Pending ───────────────────────────────────┐
    │                                         │（棚卸しで陳腐化）
    │（プランニングで取り込み）                   ▼
    ▼                                     Cancelled
  In Progress
    │
    │（改善実装完了）
    ▼
  Completed
```

---

## 使い方

1. レトロスペクティブ完了時にTryをPendingセクションに追加
2. TRY-IDは `TRY-{連番3桁}` で付与（既存の最大値+1）
3. 優先度Highのものはプランニング時に自動的にタスク候補に含める
4. タスクとして取り込んだらIn Progressに移動し、タスクIDを記録
5. 改善が実装されたらCompletedに移動し、効果を記録
6. 棚卸しトリガー（20件超過 or 10スプリント経過）到達時に棚卸しを実施
7. 陳腐化したTryはCancelledに移動し、理由を記録
8. YAMLフロントマターの集計値を更新
