# Section 9
# MCP サーバー連携

---

## MCP — 外部ツール連携の標準プロトコル

**Model Context Protocol:** AIと外部ツールを接続するオープンスタンダード

Slack投稿、GitHub Issue操作、Notion更新を **自然言語で実行可能**

**3つのトランスポート:**

| タイプ | 説明 | 用途 |
|--------|------|------|
| `http` | HTTPベースのリモートサーバー | クラウドサービス連携（推奨） |
| `stdio` | ローカルプロセス通信 | ローカルツール（Playwright等） |
| `sse` | Server-Sent Events | 非推奨。httpへ移行を |

**@メンションで直接参照:** `@github:issue://123`

---

## MCPサーバーの設定方法

```bash
# HTTPサーバー（リモート）
claude mcp add --transport http 名前 URL

# stdioサーバー（ローカル）
claude mcp add 名前 -- コマンド [引数]
```

**3スコープ:**

| スコープ | 保存先 | 共有 |
|---------|--------|------|
| `local`（デフォルト） | `~/.claude.json` | 個人 |
| `project` | `.mcp.json` | Git経由でチーム共有 |
| `user` | `~/.claude.json` | 個人・全PJ共通 |

**OAuth 2.0 認証:** `/mcp` コマンドでブラウザ認証 → トークンはKeychain保存

---

## おすすめMCPサーバー6選

| サーバー | コマンド | できること |
|---------|---------|----------|
| **Slack** | `claude mcp add --transport http slack https://mcp.slack.com/mcp` | メッセージ投稿・検索・リアクション |
| **GitHub** | `claude mcp add --transport http github https://api.githubcopilot.com/mcp/` | Issue/PR操作・コード検索 |
| **Sentry** | `claude mcp add --transport http sentry https://mcp.sentry.dev/mcp` | エラー監視・パフォーマンス |
| **Notion** | `claude mcp add --transport http notion https://mcp.notion.com/mcp` | ページ作成・DB操作 |
| **Playwright** | `claude mcp add playwright -- npx -y @playwright/mcp@latest` | ブラウザ自動操作・E2E |
| **Context7** | `claude mcp add --scope user --transport http context7 https://mcp.context7.com/mcp` | 最新ドキュメント参照 |

# カスタムスキル & エージェント

---

## カスタムスキル — /コマンドで呼べるプロンプト

繰り返し使うワークフローを `/コマンド名` で一発呼び出し

**配置:**
- `~/.claude/skills/` (Personal) / `.claude/skills/` (Project)
- **SKILL.md** ファイルで定義（YAMLフロントマター + Markdown）

```markdown
---
name: review-pr
description: PRのコード変更をレビューする
user-invocable: true
---
!`gh pr diff` の変更をレビューしてください。
観点: バグ、セキュリティ、テストカバレッジ、命名規則
```

- **動的コンテキスト:** `` !`gh pr diff` `` で実行結果を埋め込み
- **引数:** `$ARGUMENTS` で受け取り
- **context: fork** で独立コンテキストで実行

---

## SKILL.md の主要フィールド

| フィールド | 説明 | デフォルト |
|-----------|------|----------|
| `name` | スキル名 | ディレクトリ名 |
| `description` | 説明（自動呼び出し判断に使用） | 本文の最初の段落 |
| `user-invocable` | `/` メニューから呼び出し可能か | `true` |
| `allowed-tools` | 許可するツール | 全ツール |
| `model` | 実行モデル | 継承 |
| `context` | `fork` で独立コンテキスト | なし |
| `agent` | fork時のエージェント種別 | なし |

**references/ フォルダ** で補足情報を分離:
```
my-skill/
├── SKILL.md        # メイン指示
├── reference.md    # 詳細ドキュメント
└── examples.md     # 使用例
```

---

## カスタムエージェント — 専門家AIサブプロセス

特定タスクに特化した **独立サブプロセス**

**配置:** `.claude/agents/名前.md`

```markdown
---
name: code-reviewer
description: コードレビューの専門家
tools: Read, Glob, Grep
model: sonnet
maxTurns: 15
---
あなたはシニアコードレビュアーです。
```

**ツール制限（最小権限の原則）:**
- 調査専用: `Read, Glob, Grep, WebSearch`
- コーディング: `Read, Glob, Grep, Bash, Edit, Write`
- レビュー: `Read, Glob, Grep, Bash`

**組み込み:** Explore(Haiku), Plan(継承), general-purpose(全ツール)

---

## Agent Teams — 複数AIの協調（実験的）

Team Lead + Teammates で **並列タスク実行**

**有効化:** `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`

| コンポーネント | 役割 |
|-------------|------|
| Team Lead | チーム作成、タスク割り振り、進捗管理 |
| Teammates | 独立コンテキストで並列作業 |
| Task List | チーム共有のタスク管理 |
| Mailbox | チームメイト間の直接メッセージング |

**表示モード:**
- `in-process` — Shift+Down切替
- `split panes` — tmux/iTerm2で各エージェントが独立ペイン

**制限:** 1チームのみ、セッション再開不可、ネスト不可

# Section 11
# Cursorからの移行

---

## Cursorからの移行 — 基本ステップ

| 移行項目 | 手順 |
|---------|------|
| `.cursorrules` → `CLAUDE.md` | 構造化して300行以内に |
| `.cursor/rules` → `.claude/rules` | symlinkで共有可能 |
| GUI設定 → `settings.json` | 権限・Hooks・サンドボックス |
| セットアップ | `setup-coexistence.sh` で一括設定 |

**setup-coexistence.sh の効果:**
1. CLAUDE.md を Source of Truth として作成
2. `.cursorrules` → CLAUDE.md へのsymlink
3. `.claude/rules/cursor` → `.cursor/rules` からの参照

---

## タイムリープ戦略 — 使い分けが最適解

**完全移行ではなく、各ツールの強みで使い分ける**

| タスク | 推奨ツール |
|-------|----------|
| リアルタイムTab補完 | **Cursor** |
| 単一ファイルのインライン編集 | **Cursor Composer** |
| 複数ファイルリファクタ | **Claude Code** |
| テスト自動生成 | **Claude Code** |
| 外部API連携（MCP） | **Claude Code** |

**実践フロー:**
1. Cursor Composer で基本コンポーネント生成
2. Cursor Tab で型定義を高速補完
3. **Claude Code** で10ファイルのimport一括更新
4. **Claude Code** でテスト生成・Git操作

---

## 画像の扱い方

| 項目 | Cursor | Claude Code |
|------|--------|-------------|
| 方法 | チャットに貼り付け | Read toolでファイルパス指定 |
| 枚数 | 一度に1枚 | 複数画像を一度に参照可能 |
| 永続性 | セッション終了で消失 | ファイルとして永続化 |

**Claude Codeでの画像参照:**
```bash
# ペーストキャッシュ（Cmd+V）→ /tmp/claude/ に自動保存
# プロジェクト内にコピーして永続化
cp /tmp/claude/paste-123.png ./docs/assets/wireframe.png

# 参照
> ./docs/assets/wireframe.png を元にコンポーネントを実装して
```

---

## 通知システム

**macOS通知 — terminal-notifier + ccnotify.py:**
- タスク完了時にmacOSネイティブ通知を表示
- 通知クリックでVSCodeを開く
- Hookイベント: UserPromptSubmit（記録）、Stop（通知）

**Slack連携 — リアクション駆動の遠隔操作:**
```
タスク完了 → Slack投稿 → リアクション待機 → 指示変換
```
- PCから離れてもスマホでリアクション → 次の指示を自動送信
- MCP経由なら `claude mcp add --transport http slack` でトークン管理不要
