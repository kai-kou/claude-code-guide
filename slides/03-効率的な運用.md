# Section 6
# コンテキストウィンドウ管理

---

## コンテキストウィンドウの仕組み

Claudeが「覚えている情報の量」= コンテキストウィンドウ

| 要素 | 説明 | コスト |
|------|------|--------|
| Input Tokens | ユーザーの指示、Readしたファイル | 通常 |
| Output Tokens | Claudeの応答、生成コード | 通常 |
| Cache Write | 新規情報の保存 | 初回コスト発生 |
| Cache Read | 保存済み情報の再利用 | **90%割引** |

**キャッシュ有効期限:** 5分（5分以内の再アクセスで割引適用）

> キャッシュ読み取り（R）が多いほど効率的でコストが抑えられる

---

## ステータスライン — リアルタイム4行ダッシュボード

`~/.claude/settings.json` の `statusLine` 設定で有効化

```
Line 1: [PJ名] [Git:branch*] | [モデル] | [████░░ 62% WATCH] | [$0.12] | [3m42s]
Line 2: 📊 🪟 124K/200K (W:45K R:89K) | In:256K Out:142K | 残38%
Line 3: ⚡ 42% 3h12m後 | 📅 18% 5d14h後 | 🎵 38% | 🎹 15%
Line 4: 🧠 CC:142MB | App:0MB | VSCode:589MB
```

| Line | 内容 |
|------|------|
| Line 1 | プロジェクト・モデル・コンテキスト使用率・コスト・時間 |
| Line 2 | コンテキスト詳細（現在/最大, キャッシュW/R） |
| Line 3 | 利用制限（5h / 7d / Sonnet / Opus） |
| Line 4 | メモリ使用量（CLI / App / VSCode） |

---

## /compact の活用とステータスの色

**/compact:** 情報を圧縮して引き継ぐ
```
/compact Focus on the API changes, discard UI discussions
```

**コンテキスト使用率の色:**

| 色 | 範囲 | ラベル | アクション |
|----|------|--------|----------|
| 緑 | 0〜59% | OK | 安全。作業継続 |
| 黄 | 60〜79% | WATCH | 注意。/compact検討 |
| 赤 | 80〜100% | SWITCH | /compact or /clear 推奨 |

**判断の目安:**
- 黄色が見えたら「あと1〜2タスクで切り上げよう」
- タスク完了後は常に `/clear` — 最もシンプルで効果的

---

## コンテキスト管理のベストプラクティス

1. **タスク完了ごとに `/clear`**（最重要）
2. 大きなファイルは **部分読み**（offset, limitを指定）
3. 同じファイルを何度も読まない（キャッシュを活用）
4. CLAUDE.mdやよく使うファイルをセッション序盤で読み込む
5. 60%超えたら `/compact` を検討

**/clear vs /compact の使い分け:**
```
タスク完了した？
├── YES → /clear（完全リセット）
└── NO → 前の議論を引き継ぐ必要ある？
          ├── YES → /compact（要約して圧縮）
          └── NO → /clear（リセットしてやり直し）
```

# Section 7
# 利用制限の把握と対策

---

## 利用制限の種類

| 制限 | 説明 | リセット |
|------|------|---------|
| **5時間ローリング** | 直近5時間の利用量 | 常に過去5時間分 |
| **7日間制限** | 週単位の利用量 | 週単位 |
| **モデル別制限** | Sonnet/Opus それぞれの7日間制限 | 週単位 |

**プラン別:**
- Pro: 標準
- Max 5x: 5倍
- Max 20x: 20倍

**カウントルール:**
- Cache Write → 制限にカウント
- Cache Read → **90%割引**でカウント

---

## 制限の確認方法

**ステータスライン Line 3 で常時確認:**
```
⚡ 42% 3h12m後 | 📅 18% 5d14h後 | 🎵 38% | 🎹 15%
```

| 記号 | 意味 |
|------|------|
| ⚡ | 5時間制限 + リセットまでの時間 |
| 📅 | 7日間制限 + リセットまでの時間 |
| 🎵 | Sonnet使用率 |
| 🎹 | Opus使用率 |

**色分け:**
- 緑（〜70%）Safe — 安全に作業継続
- 黄（70〜85%）Caution — 軽量タスクはHaikuに
- 赤（85%〜）Danger — モデル切替 or リセット待ち

---

## 利用制限の最適化

**キャッシュヒット率を上げる（最重要）:**
- CLAUDE.md・よく使うファイルをセッション序盤で読む
- 5分以内の再アクセスでCache Read（90%割引）
- 30分以上離席するならセッションを区切る

**Haiku活用:**
- Lint、テスト、検索などの軽作業はHaikuで
- `/model haiku` で切り替え

**その他:**
- 専用ツール優先: `cat`/`grep` ではなく Read/Grep ツール
- 不要なReadを減らす: 部分読み（offset, limit）を活用
- 大きなタスクは事前に分割して複数セッションで実行

---

## 制限に近づいたら

| 使用率 | アクション |
|--------|----------|
| **50%超** | Haikuモデルに切り替え（`/model haiku`） |
| **60%超** | タスクを分割して複数セッションに |
| **80%超** | リセット待ち。緊急性の低いタスクは後回し |

**セッション分割の例:**
```
Before: 実装 → テスト → ドキュメント（1セッション）

After:
  Session 1: 実装
  Session 2: テスト + ドキュメント
```

**モデル切替:** `/model sonnet` / `/model opus` / `/model haiku`
